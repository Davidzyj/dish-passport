# 测试用例

## 构建测试

### T-001 Debug 模拟器构建

步骤：
1. 运行 `xcodebuild -project DishPassport.xcodeproj -scheme DishPassport -configuration Debug -destination 'platform=iOS Simulator,name=iPhone 17' build`

期望：
- 构建成功。
- 生成 `DishPassport.app`。

### T-002 Release 模拟器构建

步骤：
1. 运行 `xcodebuild -project DishPassport.xcodeproj -scheme DishPassport -configuration Release -destination 'generic/platform=iOS Simulator' build`

期望：
- 构建成功。
- Release 不包含可启用假数据的运行路径。

## 用户路径测试

### T-101 搜索并进入详情

步骤：
1. 打开 App。
2. 在探索页搜索 `mapo`。
3. 点击 `Mapo Tofu`。

期望：
- 搜索结果只显示匹配菜品。
- 进入麻婆豆腐详情。
- 返回后最近浏览记录更新。

### T-102 收藏闭环

步骤：
1. 进入任一菜品详情。
2. 点击“收藏”。
3. 返回探索页和收藏页。
4. 重启 App。

期望：
- 按钮变成已收藏。
- Toast 显示保存反馈。
- 收藏页“全部收藏”出现该菜。
- 重启后状态仍存在。

### T-103 想尝试闭环

步骤：
1. 进入菜品详情。
2. 点击“加入想尝试”。
3. 打开收藏页。

期望：
- 菜品出现在“想尝试”和“全部收藏”。
- 再次点击可移出想尝试。

### T-104 喜欢闭环

步骤：
1. 进入菜品详情。
2. 点击“喜欢”。
3. 打开收藏页。

期望：
- 菜品出现在“已喜欢”和“全部收藏”。
- 再次点击可取消喜欢。

### T-105 菜系批量加入

步骤：
1. 打开“菜系”。
2. 进入“川菜”。
3. 点击“全部加入想尝试”。
4. 打开收藏页。

期望：
- 川菜代表菜进入“想尝试”。
- 返回菜系列表后对应菜系显示已加入数量。

### T-106 点餐短语复制

步骤：
1. 打开短语列表。
2. 进入 `Less spicy, please.`。
3. 点击复制中文。
4. 返回短语列表。

期望：
- 复制按钮显示已复制。
- 短语进入最近列表。
- 最近状态重启后保留。

### T-107 语言选择

步骤：
1. 打开设置。
2. 选择简体中文。
3. 退出并重启 App。
4. 再选择 English。

期望：
- UI 文案立即切换。
- 重启后保留显式语言选择。
- 选择“自动”后恢复系统/地区推断。

### T-108 重置本地数据

步骤：
1. 先收藏、喜欢、加入想尝试几个菜。
2. 打开设置。
3. 点击重置收藏数据并确认。
4. 打开收藏页。

期望：
- 收藏页显示空状态。
- 最近浏览和最近短语也被清空。
- 语言选择不被清空。

## 审核与发版测试

### T-201 浅色可读性

步骤：
1. 在模拟器设置中启用深色模式。
2. 打开 App 并浏览所有页面。

期望：
- App 仍强制浅色显示。
- 浅色背景上的文字保持深色或足够对比的品牌色。
- Placeholder、禁用/次级文字不出现白字或过浅文字。

### T-202 截图假数据 Debug 模式

步骤：
1. 用 `--screenshot-demo-data` 启动 Debug App。
2. 打开收藏页。

期望：
- 看到演示收藏、想尝试、喜欢和最近浏览。
- 操作不写入生产 `UserDefaults`。

### T-203 正式启动无假数据

步骤：
1. 不带启动参数运行 Debug App 或 Release App。
2. 清空/重置本地数据后打开收藏页。

期望：
- 不出现演示数据。
- 只有用户真实操作会产生本地状态。

### T-204 发版前校验脚本

步骤：
1. 运行 `scripts/preflight_release.sh`。

期望：
- 版本、Bundle ID、iPhone-only、图标 alpha、截图模式隔离、隐私/支持网页存在性均通过。
