# Dish Passport（味游中国）UX 设计稿

版本：1.0.0 设计阶段  
目标市场：美国 App Store  
设备范围：iPhone only  
运行模式：本地 App，无账号、无后端、不主动联网  

## 产品定位

Dish Passport 是一款面向美国用户的中国菜单与饮食文化指南。用户可以在中餐馆点餐前快速查菜名、理解口味、知道怎么点，并把想尝试或喜欢的菜保存起来。它不是泛百科，而是一个“点菜时真的有用”的付费工具。

## App 名称

- 英文：Dish Passport
- 简体中文：味游中国
- 日文：中国味めぐり
- Bundle ID：com.zhouyajie.dishpassport

## 视觉方向

整体采用明确浅色主题，App 内强制浅色模式。背景为米白和暖白，但文字全部使用显式深色，不依赖系统动态颜色，避免审核设备开启深色模式时出现白字或浅灰字落在浅底上的问题。

主色建议：

- 辣椒红：#B7312C，用于主行动按钮、收藏状态、重点标签
- 墨黑：#1F2623，用于主标题和正文
- 青瓷绿：#2F6F62，用于文化、茶、地区提示
- 金橙：#D8892C，用于评分、推荐和温暖高亮
- 米白：#F7F1E6，用于主背景
- 纸白：#FFFDF8，用于内容面板

界面风格：

- 食物文化感强，但不做复古堆砌
- 内容密度适合工具类 App，首屏立即可搜索
- 图标和色块表达功能，减少说明文字
- 卡片圆角不超过 8px
- 每个页面都有清楚的主动作和返回反馈

## 信息架构

底部 Tab 共 4 个：

1. 探索：搜索菜名、浏览推荐菜、进入菜系
2. 菜系：川菜、粤菜、湘菜、江浙、北京/北方、点心、面食、甜品等分类
3. 收藏：想尝试、已喜欢、最近浏览
4. 设置：语言、隐私政策、支持、数据管理

## 核心用户路径与验收标准

### 路径 A：搜索一道菜并收藏

用户路径：

1. 用户打开 App，首页看到搜索框和热门菜品
2. 输入 “mapo” 或 “麻婆”
3. 搜索结果实时筛选出 “Mapo Tofu / 麻婆豆腐”
4. 点击结果进入详情页
5. 看到菜名、拼音、辣度、主要食材、地区、点菜短语和文化背景
6. 点击收藏按钮
7. 按钮变为已收藏，并显示短反馈
8. 返回首页或进入收藏页，能看到刚收藏的菜

验收标准：

- 搜索输入后列表确实变化
- 点击搜索结果进入正确详情
- 收藏状态保存到本地
- 返回后首页卡片显示已收藏状态
- 收藏页显示该菜
- 重启 App 后收藏仍存在
- SwiftUI 中收藏字典/数组变更必须重新赋值或通过可观察模型触发 UI 刷新

### 路径 B：浏览菜系并保存想尝试

用户路径：

1. 用户进入“菜系”
2. 点击 “Sichuan / 川菜”
3. 看到该菜系的口味说明、代表菜和点餐建议
4. 点击 “Add to Try List”
5. 该菜加入“想尝试”
6. 返回“收藏”，在“想尝试”分组可见

验收标准：

- 菜系入口清楚
- 菜系列表有真实内容，不是空入口
- 保存后有按钮状态变化和短反馈
- 收藏页按状态分组展示
- 重启后状态保留

### 路径 C：学习一句点餐短语

用户路径：

1. 用户在首页点击“点餐短语”
2. 进入短语列表
3. 选择“Less spicy, please.”
4. 查看英文、中文、拼音、使用场景
5. 点击复制
6. 按钮显示已复制反馈
7. 该短语进入最近查看

验收标准：

- 复制动作有效
- 最近查看状态保存
- 返回列表后该短语显示最近标识
- 不需要网络

### 路径 D：切换语言

用户路径：

1. 用户进入设置
2. 点击语言
3. 选择 English / 简体中文 / 日本語
4. App 立即按选择切换主要 UI 文案
5. 重启后仍保持用户选择

验收标准：

- 显式选择优先级高于系统/地区推断
- 没有显式选择时按地区/系统推断，兜底英文
- CFBundleDisplayName 配置英文、中文、日文
- 设置页不显示 Bundle ID

### 路径 E：打开隐私政策和支持

用户路径：

1. 用户进入设置
2. 点击隐私政策或支持
3. 通过系统浏览器打开 GitHub Pages 对应页面
4. 页面支持英文、中文、日文

验收标准：

- App 内入口存在
- 链接可打开
- 网页三语内容完整
- App 本身不主动发起网络请求，只有用户显式点击外部链接

## 数据设计初稿

本地内置内容：

- 100 道常见中国菜
- 8 个菜系/分类
- 30 条点餐短语
- 20 条文化小卡片

用户本地数据：

- 收藏菜品 ID
- 想尝试菜品 ID
- 已喜欢菜品 ID
- 最近浏览菜品 ID 和短语 ID
- 语言选择
- 首次启动状态

推荐存储：

- 内置内容使用 JSON bundled resource
- 用户状态使用 Codable + UserDefaults 或本地 JSON 文件
- 收藏等集合在 SwiftUI 中通过 ObservableObject 管理，变更时重新赋值，确保 UI 刷新

## 截图专用假数据模式

后续实现必须包含独立的截图演示数据模式：

- 只通过显式启动参数启用，例如 `--screenshot-demo-data`
- Debug/截图脚本可使用
- Release 正式启动不得初始化、显示、持久化任何假数据
- 假数据模块与生产数据模块隔离
- 提供截图脚本
- 提供发版前校验脚本，检查 Release 配置不会启用假数据
