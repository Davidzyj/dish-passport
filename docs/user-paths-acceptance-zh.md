# 用户路径与验收标准

## 1. 探索与搜索

用户路径：
1. 用户打开 App，默认进入“探索”页。
2. 用户在搜索框输入英文、拼音或中文菜名。
3. 列表实时显示本地结果；点任意菜品进入详情。
4. 返回探索页后，已收藏菜品在列表中显示状态。

验收标准：
- 搜索不发起网络请求。
- 键盘可通过 Done、工具栏按钮或滚动自然收起。
- 空结果有明确提示。
- 进入详情会记录最近浏览，并持久化到本机。

## 2. 菜品详情

用户路径：
1. 用户从探索、菜系或收藏进入菜品详情。
2. 用户查看辣度、地区、风味、食材、点餐表达和文化说明。
3. 用户点击收藏、想尝试、喜欢。
4. 用户返回来源页面，状态图标与收藏列表同步变化。

验收标准：
- 三个操作均有真实状态变更和 toast 反馈。
- 状态保存在设备本地，重启 App 后仍然存在。
- `@Published` 中的集合变更通过重新赋值触发 SwiftUI 刷新。
- 浅色背景上的文字全部使用显式深色或足够对比的品牌色。

## 3. 菜系页

用户路径：
1. 用户进入“菜系”页浏览地区风味。
2. 用户点一个菜系进入详情页。
3. 用户点击“加入想尝试”批量保存代表菜，或点单个代表菜进入详情。
4. 返回菜系页或收藏页时能看到状态变化。

验收标准：
- 每个菜系有说明、点餐建议和代表菜。
- 批量加入会保存到本地并给出反馈。
- 已加入的代表菜在列表中显示已保存状态。

## 4. 点餐短语

用户路径：
1. 用户从探索页“点餐短语”入口或短语列表进入短语页。
2. 用户查看短语、拼音和使用场景。
3. 用户复制中文短语。
4. 最近使用短语区域更新，并在重启后保留。

验收标准：
- 复制按钮可用，有复制成功反馈。
- 最近短语状态本地保存。
- 页面支持英文、简体中文、日语界面。

## 5. 收藏页

用户路径：
1. 用户进入“收藏”页。
2. 用户查看“想尝试”“喜欢”“全部收藏”“最近浏览”。
3. 用户点任一菜品进入详情并修改状态。
4. 返回收藏页后分区内容同步更新。

验收标准：
- 无收藏时显示明确空状态。
- 各分区只显示当前本地状态中的菜品。
- 从详情返回后列表立即刷新。

## 6. 设置页

用户路径：
1. 用户进入“设置”页。
2. 用户显式选择英文、简体中文或日语。
3. 用户打开隐私政策或支持网页。
4. 用户可确认后重置本地收藏数据。

验收标准：
- 显式语言选择持久化；未选择时按系统/地区推断，兜底英文。
- 设置页不显示 Bundle ID。
- 隐私/支持入口只有点击时才打开外部网页。
- 重置操作有确认弹窗和完成反馈。

## 7. 截图专用假数据模式

用户路径：
1. 截图脚本以显式启动参数 `--screenshot-demo-data` 启动 Debug App。
2. App 显示演示收藏、最近浏览和短语状态，用于 App Store 截图。
3. 正式启动和 Release 构建不初始化、不显示、不持久化任何假数据。

验收标准：
- 假数据逻辑只在 `#if DEBUG` 下编译。
- 只有显式启动参数能启用截图数据。
- 截图模式不写入 `UserDefaults`。
- 发版前校验脚本能检查 Bundle ID、版本号、iPhone-only、截图模式隔离和图标 alpha。
